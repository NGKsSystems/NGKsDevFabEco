================================================================================
QUICKSTART: Get DevFabEco Running in 5 Minutes
================================================================================

PREREQUISITES (read PREREQS.txt for details):
□ Python 3.11+ installed
□ Visual Studio / MSVC installed (for C++ projects)
□ git installed
□ PowerShell 5.0+ (or PowerShell Core)

IF ANY ARE MISSING: Read PREREQS.txt before proceeding.

STEP 1: Verify Prerequisites (2 min)
────────────────────────────────────
cd dist\devfabeco_release_bundle
.\tools\check_prereqs.ps1

Fix any issues, then continue.

STEP 2: Run the Bundle (1-5 min depending on project size)
───────────────────────────────────────────────────────────
.\run.ps1

You will see:
- Status messages as the workflow runs
- Proof folder location
- Final PASS or FAIL decision
- Dashboard file path

STEP 3: Check Results (2 min)
──────────────────────────────
Open in your browser:
  status\dashboard.html

This shows:
- PASS or FAIL gate decision
- Link to full proof artifacts
- Why it passed or failed
- What to do next

STEP 4: Interpret the Result
──────────────────────────────
READ: docs\GATE_SEMANTICS.txt

In short:
  ✓ PASS = Safe to ship, all gates cleared
  ✗ FAIL = Stop, certification failed, debug required

COMMON NEXT STEPS:

If PASS:
  → Review what was proven in the dashboard
  → Proceed with your deployment process
  → Archive the proof folder if needed

If FAIL:
  → Read the failure summary in the dashboard
  → Check FAILURE_GUIDE.txt for common issues
  → Read the detailed proof artifacts in _proof/<run_id>/
  → Fix the underlying issue and re-run

THAT'S IT!
──────────
The bundle handles all the complexity. The dashboard tells you if you're 
ready to ship or if you need to debug further.

For more details, see:
- README.txt (overview)
- PREREQS.txt (system requirements)
- docs/GATE_SEMANTICS.txt (what PASS means)
- docs/CUSTODY_POLICY.txt (guarantees)

Questions? See FAILURE_GUIDE.txt.
